Introduction

Java is class based object orientated programming language, where the code you write is designed to run on all platforms that support java. The code must be compiled to run, and is compiled to byte code that can run on any JVM. Java syntax is similar to c/c++. The JVM works on three principles,

  • Write code - this is written in an IDE/text-editor, written by a human.
  • Compile code - compilation of code is done by JAVAC; included in JDK(java development kit); which takes Java program as input and outputs a bytecode.
  • Execute code - In the running phase, the JVM executes this bytecode generated by the compiler.
Every operating system follows this process, essentially everything is run at bytecode level with their own JVM. Because of this, java is known as platform-independent language.

For any java program to run, a computer must have a JRE(JAVA RUNTIME ENVIRONMENT); this won't compile the code, but it includes a browser, JVM, applet supports and plugs.

Java has a built in program called Garbage Collector, where non referenced objects are collected. Rather than humans manually managing this, it is managed by the programmed. Where its main objective is to free heap memorby by destroying unreachable objects.

In Java, there are many libraries with useful framework which can be used to help build your programs. For this reason we have classpath, where java runtime and java compiler look for .class files to load.

For example,

import org.company.Menu

This import statement makes the Menu class available to use in our current class, such that the Menu can be instantiated.

Menu menu = new Menu()

By importing, and specifying the full package, the JVM will know where to find Menu class, and be used in our code.

Java vs Javascript

Though they share similar name, syntactically and someways sometimes similar, fundamentally they are very different. JS(java script) resembles Java but does not have Java's static typing and strong type checking. JS follows Java's expression syntax, naming conventions and basic control-flow constructs which was the reason it was actually renamed from LiveScript to JavaScript.

Java is not as free-form language as JS. In Java you must declare all variables, classes and methods. Methods also need to be declared either public, private or protected. Interfaces can also sometimes be implemented and often recommended. Variables, parameters and functions also need a return type explictly unlike in JS.

Basic OOP Principles

Java is an OOP Language, what this means is that it is an Object Orientated Programming language; where a program is in terms of a collection of objects, each of which is comprised of data and code. Where data is in the form of fields otherwise known as attributes or properties and code, in the form of procedures also known as methods. In Java, these OOP objects are class based, where objects are instances of classes. In OOP, we have 4 main concepts:

  • Abstraction - Without delving too deep into the code, derived objects will only reveal necessary information required. If needed, functionality can be extended from these derived classes, allowing developers to make additions to functions or methods.
  • Encapsulation - An object will have all information associated with it, contained within its class; similar to abstraction, only select information will be exposed. These implementations are privately held inside the defined class and other objects that don't have access to this class won't be able to modify or make changes. These objects will only be able to call functions or methods that are listed public.
  • Inheritance - This allows for classes to reuse codes from other classes. This allows developers to maintain a system of hierarchy, allowing for better reusability of code, lowering development time and reduces need for duplicates.
  • Polymorphism - Is the ability for an object to share similar behaviours to others. Based on parent class, the program will determine how the object will behave based on the

Main benefits of OOP languages are:

  • Code reusability
  • Scalability
  • Efficiency

Hello World

Due to the portability of Java, if you have JVM installed on one machine it can be run on another machine. To write the most basic code, to know we have Java working in our machines. We will print out on console, hello world 


//Basic java hello world program to print "hello world"
//import modules/libraries required
  import java.io*
   
  // set up class 
  class HelloWorld {
  public static void main(String[] args) {
    
    // prints Hello World
    System.out.println("Hello World");
  }
    
}

Let's break down what's happening,

  • Comments - first few lines are known as comments and used to explain the code, for this example we have commented for every new line, but for general practice, you don't need to comment for every single line. Good practice would be to comment at the start explaining what the code does, and comment the functionalities of any methods
  • Import - second line imports from java.io*, this is essentially importing all classes of the io package. This io package provides a set of input and output streams for reading/writing data files. We import classes to provide us with methods/functionalities to use within our code
  • Class - The class contains the data and methods to be used in the program. Methods define the behaviour of a program, in this example we only have one method called main, which is void, meaning it does not request anything to be returned
  • Within the main method, System.out.println("Hello World"), this calls the method println() from the standard output stream which prints whatever is within the method onto console.

Variable Scope

Scopes are the region where a certain variable is accessible within a region of the program. Depending on the location they are created, that variable is only accessible by a function or method after it has been declared.

In Java we have local scope, sometimes known as a method level scope where a variable is declared within a method or function and the variable is NOT accessible outside this method. A global scope or sometimes known as Class level scope, are variables declared within a class usually at the very top of the code execution block and is accessible to all methods within that class - depending on whether it is public or private it can even be accessed outside the class. A Block scope are sometimes declared in for loop condition and is not accessible after the loop is complete; unless defined seperately beforehand.
